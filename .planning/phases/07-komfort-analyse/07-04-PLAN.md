<!-- Contracts from 07-01 + codebase. -->

From 07-01 dice-stats-idb.js: `window.getAllStats()` → Promise<record[]>, `window.getStatsForSession(sessionId)` → Promise<record[]>, `window._currentSessionId` (app-boot string). Record shape written by the hook: `{ id, notation, result, rolls, timestamp, sessionId, charId }`.

Attribution (D-05): character-attributed rolls embed the character name as a `"Name: "` prefix in `notation` (e.g. `rollAttrCheck` → `"Thorin: STR"`, `rollCharSave` → `"Thorin: STR Save"`, `rollSkillCheck`, `"<name> Init"`). The hook stores `charId: null` for now, so attribution is derived by parsing the `notation` prefix before `": "`/`" "`. Rolls with no recognizable character prefix (Dice-tab, quick-roll R, Vorteil/Nachteil, generic attack) → "Allgemein". Match the parsed name against `D.characters[].name` to group; unknown names also fall to "Allgemein".

d20 identification (RESEARCH § "Computing d20 Histogram Counts"): a record contributes to the d20 histogram when `notation` includes `d20`/`D20` or equals `Vorteil`/`Nachteil`, and each face value in `rolls` is an int 1–20. `computeD20Counts(records)` → `Array(20)` where index i = count of face (i+1).

07-01 created `#dicestats-container` (view-tools.html) + `renderDiceStats()` skeleton; tab registered as `dicestats`. esc()/showToast global. CSS theme vars: `var(--green)` (crit), `var(--red)` (fumble), `var(--blue)` (bars), `var(--gold)` (overlay), `var(--text-dim)`.
