<objective>
Build the Soundboard engine: persist audio files as Blobs in IndexedDB (D-01) with a size guard (D-01a), and play them via the Web Audio API as layered looping tracks with per-track volume and crossfade on scene switch (D-02, full).

Purpose: This is the audio core both the Library UI and scene UI (07-03) sit on top of. It is greenfield Web Audio code — `new Audio()` cannot do layered loops with independent gain, so the Web Audio node graph is mandatory.

Output: `soundboard-idb.js` (blob persistence + size guard) and `soundboard-player.js` (AudioContext lifecycle, decode/loop/gain, scene activation with crossfade).

**D-02 vs D-02a decision — IMPLEMENT D-02 (full layered crossfade).** Rationale: the user's own ambience example (Regen + Taverne + Feuer) is layered by definition, and per RESEARCH § Open Question 1 the code delta over D-02a is modest — D-02a (one loop, hard cut) is a strict subset (N=1 source, no gain ramp). The crossfade is `GainNode.gain.linearRampToValueAtTime()` over `CROSSFADE_DURATION`. The D-02a hard-cut path is NOT used; it is documented here only as the considered-and-rejected fallback.
</objective>

IDB store created in 07-01 (core/init.js, v4): `audioBlobs` keyPath `id`. Access via `window.idb` after `await window.initIndexedDB()`.

Proven IDB tx pattern (systems/backups.js:88 / :115):
- write: `const tx = window.idb.transaction(['store'],'readwrite'); tx.objectStore('store').put(entry); tx.oncomplete = resolve; tx.onerror = reject;`
- read all: `store.getAll()` → `req.onsuccess = () => resolve(req.result||[])`.

Web Audio pipeline (RESEARCH Pattern 2/3, MDN-cited):
- `blob.arrayBuffer()` → `ctx.decodeAudioData(arrayBuffer)` → `AudioBuffer`.
- `ctx.createBufferSource()`, `.buffer = audioBuffer`, `.loop = true`, `.connect(gainNode)`, `gainNode.connect(ctx.destination)`, `.start(0)`. Source nodes are one-shot — create a NEW source per play (RESEARCH Pitfall 2).
- crossfade: `gain.gain.setValueAtTime(v, now); gain.gain.linearRampToValueAtTime(target, now + CROSSFADE_DURATION);` then `oldSource.stop(now + CROSSFADE_DURATION + 0.1)`.

esc()/sanitizeHTML() are global. showToast(msg, type) is global.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file picker → IDB | Untrusted local audio file (name, type, size, bytes) crosses into IDB and the audio decoder |
| IDB Blob → Web Audio decoder | Stored bytes are decoded by `decodeAudioData` — crafted/corrupt files can throw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-AUDIO-DOS | Denial of Service | saveSoundBlob IDB write | mitigate | `checkAudioFileSize` + hard block at >100 MB before any IDB write (D-01a); soft warn at >20 MB. Covered by the "size warning" unit test (Task 1). |
| T-07-AUDIO-DECODE | Tampering | decodeAudioData on stored Blob | mitigate | try/catch around decodeAudioData + `showToast('Audio konnte nicht gelesen werden','error')`; never throws uncaught (Task 2) |
| T-07-AUDIO-NAME | Spoofing | file.name stored in audioBlobs | mitigate | Name is stored raw here but MUST be `esc(file.name)` when rendered in the Library UI — obligation passed to 07-03 (no HTML render in this plan) |
| T-07-DEDUP | Tampering | global-scope audio context | mitigate | Uniquely-named `_soundboardAudioContext` (not `audioContext`), no `var X = window.X` of a const-global; clean build verified (Pitfall 5) |
| T-07-SC | Tampering | npm installs | accept | No package installs — native Web Audio + IDB only (RESEARCH § Package Legitimacy Audit) |
</threat_model>
