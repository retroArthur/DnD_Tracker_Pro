## Typography

All sizes are inherited from the existing design system. No new font sizes introduced.

| Role | Size | Weight | Line Height | Usage in Phase 4 |
|------|------|--------|-------------|------------------|
| Body | 14px (desktop) / 16px (mobile) | 400 | 1.5 | Drawer statblock body text, trait descriptions |
| Label | 12px (`var(--font-size-small)`) | 600 | 1.2 | Pip labels (`LA`, `LW`), mob type sub-label, drawer section headers |
| Heading | ~1.1em of base (`var(--font-size-title)`) | 700 | 1.2 | Drawer panel header (combatant name), mob row name |
| Display | Not used | — | — | — |

> **Accepted deviation — 3 font weights (400 / 600 / 700).** This exceeds the generic "max 2 weights" UI guideline. It is intentional and required: decision **D-08** locks the new pip labels to the *exact* look of the existing Death-Save / Concentration elements, which use `font-weight: 600` (`initiative.css:277`), while combatant names use `700` (`initiative.css:146`) and the round badge `700` (`initiative.css:21`), and body text `400`. Forcing 2 weights would make Phase-4 labels visibly heavier or lighter than the adjacent existing initiative UI. See **## Accepted Deviations**. (User-approved 2026-06-13.)

**Specific sizes used by new elements:**
- `.la-label`, `.lr-label`: `font-size: 0.75em; font-weight: 600` — matches `.death-saves-label` (initiative.css:278).
- `.init-type` (mob sub-label): `font-size: 0.75em; color: var(--text-dim)` — inherited from existing `.init-type`.
- Statblock parchment text in drawer: inherits from `.bestiary-statblock` styles (existing bestiary CSS).
- Mob alive count ("7 von 10 am Leben"): `font-size: 0.85em; color: var(--text-dim)` — matches pattern of `.init-type`.

## Component Visual Contracts

### INIT-01: Statblock-Drawer

**Focal point:** On open, the eye lands on the parchment statblock header — the creature name (heading, weight 700) above the rule divider — at the top of the panel. For combatants without a `statblockRef`, the focal point is the name + HP/AC block of the basic-info card.

**Trigger:** `📖` button rendered inside each `.init-entry` row, right-aligned in the controls cluster.

**Button spec:**
```
element:    <button class="btn-icon init-statblock-btn" data-action="show-init-statblock" data-id="{cb.id}" title="{tooltip}">📖</button>
size:       24px × 24px (same as existing init icon buttons — HP ±, effect)
placement:  Appended to `.init-right` flex container, after existing HP and action buttons
```

**Drawer spec:**
```
element ID:   init-statblock-panel
class:        modal-overlay init-statblock-drawer
position:     fixed, right: 0, top: 0, height: 100%
width:        420px (desktop) / 100vw (mobile viewport < 600px)
background:   var(--bg-dark) for content panel interior
shadow:       box-shadow: -4px 0 24px rgba(0,0,0,0.5) on content panel
overflow-y:   scroll on content panel
z-index:      1000 (matches modal-overlay convention)
open class:   .show (toggles display: flex)
```

**Drawer content — with statblockRef (Bestiary-Monster):**
- Full parchment statblock via `renderStatblockHTML(monster, source)` — same visual as `.bestiary-statblock` in the Bestiary tab.
- Reuses `.read-aloud.parchment` class for the warm parchment aesthetic.
- Clickable dice spans: `<span class="bestiary-dice" data-action="bestiary-roll-dice" ...>` — existing handler reused verbatim.
- Render order: `sanitizeHTML(rawText)` then `renderClickableDice(sanitizedText)` — mandatory order per Phase 3 decision.

**Drawer content — without statblockRef (Spieler / manuell):**
```
element:  <div class="init-statblock-basic">
shows:    Name (heading), HP current/max, AC, active effects list, active conditions list
style:    card-like within drawer, no parchment background — plain bg-card
```

**Overlay close:** Click outside `.init-statblock-content` (on `.modal-overlay` dim) closes drawer.

**Keyboard:** Escape key closes drawer (existing ESC handler via overlay pattern).

**Responsive (mobile < 600px):**
```
position: fixed, bottom: 0, top: auto, left: 0, right: 0
width:    100vw
height:   60vh
border-radius: 12px 12px 0 0 (bottom-sheet)
```

### INIT-02: Legendäre Aktionen — Pip UI

**Placement:** Rendered as a block inside the `.init-entry` detail area, directly below the combatant name row. Appears only when `cb.legendaryActions && cb.legendaryActions.max > 0`.

**HTML structure (matches `.death-saves` pattern exactly):**
```html
<div class="la-pips">
  <span class="la-label">⭐ LA</span>
  <div class="la-dots">
    <!-- N dots, N = cb.legendaryActions.max -->
    <span class="la-dot [active]"
          data-action="init-use-la-stop"
          data-id="{cb.id}"
          data-index="{i}"
          title="Legendäre Aktion {i+1} verwenden"></span>
    ...
  </div>
  <!-- title on container div explains auto-reset -->
</div>
```

**Pip dot states:**
- `active` (remaining > index): filled circle, background `var(--gold)`, border `var(--gold)`
- default (used): empty circle, border `var(--gold)`, background transparent

**Click behavior:** Clicking any filled (active) pip removes the topmost remaining pip (decrement `remaining` by 1). Clicking an empty pip re-fills from current count (increment `remaining` by 1 if index === remaining). This mirrors Death Save toggle pattern.

**CSS spec:**
```css

    gap: 8px;
    margin-top: 4px;

```

**Auto-reset trigger:** LA reset occurs silently inside `nextTurn()` when `init.currentTurn` wraps back to 0 (new round). No toast shown (matches lair-action convention). Comment in code: `// D-10: LA-Reset bei Rundenübergang`.

## Feature Hiding: Mob Row vs. Single-Combatant Features

This is a mandatory interaction contract. The mob row MUST hide or suppress features designed for single combatants.

| Feature | Mob Row Behavior | Implementation Note |
|---------|-----------------|---------------------|
| Death Saves | HIDDEN — already safe | `renderDeathSaves()` only triggers for `type === 'player'`. Mob has `type === 'monster'`. No code change needed. |
| Concentration Badge | HIDDEN | Guard: `if (!dead && !cb.mob) renderConcentration(cb)` |
| Concentration Check Banner | HIDDEN | Guard: `if (!cb.mob && cb.concentration?.pendingCheck)` |
| AoE-Modal Target | SHOWN as single target | Mob appears as one target in AoE modal. Damage hits `cb.currentHp` (Pool-HP). `modHp()` call is already correct. |
| Quick Actions Bar | HIDDEN | Guard: `if (!cb.mob) renderQuickActionsBar()` — Dodge/Dash/Hide are meaningless for a mob group. |
| Spell Slots | HIDDEN — already safe | Spell slots only rendered for `type === 'player'`. No code change needed. |
| HP +/- Buttons | SHOWN — modifies Pool-HP | No change. HP buttons operate on `cb.currentHp` which is the Pool-HP. |

## Accepted Deviations

These deviate from the generic GSD UI dimension rules but are **deliberate and user-approved (2026-06-13)**, because Phase 4 extends a mature in-house design system rather than a greenfield one. Consistency with the existing initiative module (locked decision **D-08** — "exakte Optik" of the existing Death-Save / Concentration elements) takes precedence over the generic defaults.

| Dimension | Generic rule | This spec | Justification (code evidence) |
|-----------|-------------|-----------|-------------------------------|
| Typography | ≤ 2 font weights | 3 weights: **400 / 600 / 700** | The existing `initiative.css` already uses 500/600/700 throughout. New pip labels MUST match the Death-Save label `font-weight: 600` (`initiative.css:277`); headings / combatant names use `700` (`:21`, `:146`); body `400`. D-08 locks pip optik to these neighbours — collapsing to 2 weights would make the new labels visibly mismatched against the elements directly beside them. |

**Spacing was NOT deviated.** Per the user's choice, all new mob-control spacing was normalised onto the 4px grid (`gap`/`padding`/`margin` values: 2px→4px, 3px→4px, 6px→8px, 10px→8px). The pip CSS (`.la-dot`/`.lr-dot`, 16px / `gap:4px` / `gap:8px` / `margin-top:4px`) was already grid-conformant. See **## Spacing Scale**.
