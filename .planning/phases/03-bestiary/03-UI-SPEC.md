# Phase 3 — Bestiary — UI Design Contract

> Visual and interaction contract for the Bestiary tab (Phase 3).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **BROWNFIELD NOTE:** This is an existing app with a complete CSS design system.
> Every token below is REUSED from `assets/styles/variables.css` and existing CSS files.
> No new color palette, font stack, or spacing system is introduced.
> The new `assets/styles/bestiary.css` uses CSS custom properties already declared in variables.css.

## Design System

| Property | Value | Source |

## Spacing Scale

Declared values (multiples of 4 only). All tokens map to CSS custom properties already
present on `:root` via the existing `--content-padding: 16px` and `--button-padding` variables.

| Token | Value | Usage in Bestiary |
|-------|-------|-------------------|
| xs | 4px | Icon-to-text gaps in list row; badge internal padding (4px 8px) |
| sm | 8px | Gap between filter chips; gap between list items; toolbar element gaps |
| sm+ | 12px | Toolbar `padding-top`/`padding-bottom`; list item `padding-top`/`padding-bottom`; `gap` inside statblock action list |
| md | 16px | Toolbar padding (horizontal); detail panel content padding (horizontal); list master padding; list item `padding-left`/`padding-right` |
| lg | 24px | Statblock section vertical margin (between Aktionen / Eigenschaften blocks) |
| xl | 32px | Statblock header bottom margin |
| 2xl | 48px | Bottom padding of list column (scroll clearance for add button) |
| 3xl | 64px | — (not used in this phase) |

Exceptions:
- Touch targets (buttons, list items, filter chips on mobile): minimum 44px height — enforced via `[data-layout="mobile"] .btn { min-height: 44px }` already in variables.css.
- Statblock attribute row cells: fixed 52px wide to align 6-column grid; not on the 4px grid but dictated by D&D 5e statblock convention.
- Favoriten-Stern hit target within list row: 32×32px tap zone (inline element expanded via padding).
- Search input right padding: 30px functional icon-clearance offset (not a layout spacing token — accepted by checker).

## Typography

All sizes use the existing `--font-size-base: 14px` (desktop) / `16px` (mobile) variable system.
No new font is introduced. The app already loads Google Fonts (Roboto, Inter, Poppins, Source Sans Pro) via build.py.

**Exactly two font weights are used across this entire phase: 400 (regular) and 700 (bold).**

| Role | Size | Weight | Line Height | Usage in Bestiary |
|------|------|--------|-------------|-------------------|
| Body | 14px (`var(--font-size-base)`) | 400 (regular) | 1.5 | List item meta text (CR, Typ); filter labels; detail panel prose sections (Traits, Aktionen, Reaktionen) |
| Label | 12px (`var(--font-size-small)`) | 400 (regular) | 1.4 | Badges (SRD / Eigen); statblock sub-labels (Rettungswürfe:, Sinne:, Sprachen:); filter chip text |
| Heading | ~16px (`1.1em` relative to base) | 700 (bold) | 1.2 | Statblock section headings (Eigenschaften, Aktionen, Reaktionen, Legendäre Aktionen); toolbar title |
| Display | ~18px (`1.2–1.3em` relative to base) | 700 (bold) | 1.1 | Monster name in detail panel header (maps to `.npc-detail-name` pattern: `font-size: 1.2em; font-weight: 700`) |

Statblock body text (inside the parchment panel):
- Font: `Georgia, 'Times New Roman', serif` — reused from `.read-aloud` rule in party.css
- Size: 13–14px, Weight 400, Line-height 1.7 (matches existing `.read-aloud` line-height)
- Statblock monster-name heading inside parchment: 18px, weight 700, `color: var(--red)` (classic 5e red)
- Statblock sub-heading (size/type/alignment italic line): 12px, weight 400, italic, `color: var(--text-dim)`

CR display format in statblock: `Herausforderungsgrad 1/4 (50 EP)` — full label from RESEARCH.md UI-text table.
CR display in list row: `HG 1/4` — abbreviated label.
CR fractions (0, 1/8, 1/4, 1/2) are stored as strings and displayed verbatim (never converted to decimals for display).

## Color

All values are CSS custom property references — behavior adapts to the 4 existing themes
(dark / light / sepia / high-contrast). No hardcoded hex values in bestiary.css except inside
parchment block overrides (which are theme-independent by design, reusing existing read-aloud values).

| Role | CSS Variable | Light/Dark default | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `var(--bg-dark)` | `#0d0d0d` / `#f5f5f5` | List master column background; tab view background |
| Secondary (30%) | `var(--bg-card)` | `#1a1a1a` / `#ffffff` | Detail panel background; toolbar background; list item cards; editor modal background |
| Elevated surface | `var(--bg-elevated)` | `#252525` / `#e8e8e8` | Selected list item; hovered list item; attribute table cells in statblock |
| Accent (10%) | `var(--gold)` | `#d4af37` / `#b8860b` | **Reserved for:** active/selected list item border, toolbar title text, filter chip active state, monster name in detail header, statblock red headings fallback on light theme, favoriten-stern active state |
| Destructive | `var(--red)` | `#ef4444` / `#dc2626` | Delete button; statblock section headings (Eigenschaften / Aktionen / etc. — classic 5e red); delete confirmation dialog |
| Parchment background | `linear-gradient(135deg, #f5e6c8 0%, #e8d4a8 100%)` | (theme-independent) | Statblock parchment panel background — reused from `.read-aloud.parchment` rule |
| Parchment border | `#8b4513` | (theme-independent) | Statblock parchment left-border accent |
| Parchment text | `#2c1810` | (theme-independent) | Text inside statblock parchment block |

Accent reserved for (explicit list):
1. Selected list item left border / outline
2. Toolbar title ("Bestiar") text color
3. Active filter chip background (chip turns `var(--gold)` with `var(--bg-dark)` text — same as `.npc-filter-chip.active`)
4. Monster name in detail header
5. Favoriten-Stern in active state (`★` gold)
6. Hover border on list items and toolbar buttons

Second semantic color (destructive — `var(--red)`):
- Statblock section heading text color (classic 5e layout)
- Delete button and confirmation dialog
- Error validation message in editor ("⚠️ Name erforderlich — bitte einen Namen eingeben")

Badge colors:
- SRD badge: `var(--bg-elevated)` background, `var(--text-dim)` text — subtle/read-only origin signal
- Eigen badge: `var(--gold-dim)` background, `var(--bg-dark)` text — distinguishes user-created content

| "Zur Initiative" button | `.btn.btn-primary` | Label: "Zur Initiative"; `background: var(--gold); color: var(--bg-dark)` |
| "Zu Encounter" button | `.btn` | Label: "Zu Encounter"; outline style |

| Attribution footer | `.bestiary-attribution` | `font-size: 0.75em; color: var(--text-dim); margin-top: 16px; padding-top: 8px; border-top: 1px solid var(--border)` |

**Statblock section ordering (inside parchment):**
1. Monster name (`font-size: 18px; font-weight: 700; color: var(--red)`)
2. Size / Typ / Alignment italic subline (`font-size: 12px; font-style: italic; color: var(--text-dim)` within parchment = `rgba(44,24,16,0.7)`)
3. Horizontal rule (1px, `border-color: #8b4513`)
4. AC / HP / Speed row
5. Horizontal rule
6. Attribute row (STR / DEX / CON / INT / WIS / CHA) — 6-column grid, 52px cells, modifier in parentheses
7. Horizontal rule
8. Rettungswürfe (if any)
9. Fertigkeiten (if any)
10. Schadenswiderstände (if any)
11. Schadensimmunitäten (if any)
12. Zustandsimmunitäten (if any)
13. Sinne
14. Sprachen
15. Herausforderungsgrad X (Y EP)
16. Horizontal rule
17. Eigenschaften (traits — if any)
18. Aktionen
19. Reaktionen (if any)
20. Legendäre Aktionen (if any) — bold red heading, intro sentence before action list

Section headings (Eigenschaften, Aktionen, etc.): `font-size: 16px; font-weight: 700; color: var(--red); font-family: Georgia, serif; border-bottom: 1px solid var(--red); margin: 12px 0 6px 0`.
Horizontal rules between sections: `<hr>` with `border: none; border-top: 1px solid #8b4513; margin: 8px 0`.

**Clickable dice (D-09):**
- To-hit modifiers (`+4`): wrapped in `<span class="bestiary-dice" data-action="bestiary-roll-dice" data-value="1d20+4">`. Visual: `color: var(--cyan); cursor: pointer; text-decoration: underline dotted`.
- Damage formulas (`1d6+2`): wrapped in `<span class="bestiary-dice" data-action="bestiary-roll-dice" data-value="1d6+2">`. Same visual treatment.
- Hover: `color: var(--gold)`.

### 4. Quantity Dialog ("Zur Initiative")
Pattern: Uses existing `prompt()` browser dialog as specified in RESEARCH.md Muster 5.

| State | Behavior |
|-------|----------|
| Trigger | Click "Zur Initiative" button in detail panel |
| Dialog | `prompt("Wie viele \"%name%\" zur Initiative hinzufügen?", "1")` |
| Validation | `Math.max(1, parseInt(input) || 1)` — silently clamps; no separate error |
| Cancel | `prompt()` returns `null` → operation aborted, no toast |

### 5. Kreatur-Editor Modal (BEST-02)
Pattern: Reuses `.modal` structure from existing modal system. Own `bst-*` field IDs to avoid collision with `enc-*` encounter form fields.

| Element | CSS Class / ID | Token/Behavior |
|---------|---------------|----------------|
| Modal container | `.modal` | Existing modal CSS; `--modal-width: 500px` (desktop) |
| Modal title (new) | `#bestiary-editor-title` | "Neue Kreatur" |
| Modal title (edit) | `#bestiary-editor-title` | "Kreatur bearbeiten" |
| Portrait field (optional) | Avatar button via `showAvatarModal()` | Only shown for custom creatures |
| Form fields | `#bst-name`, `#bst-cr`, `#bst-type`, `#bst-size`, `#bst-alignment`, `#bst-ac`, `#bst-hp`, etc. | `bst-*` prefix; mirrors encounter form structure |
| Rich-text fields | Traits, Aktionen, Reaktionen, Legendäre Aktionen | Reuse floating rich-text toolbar (globally available) |
| Validation error | Inline below name field | `color: var(--red); font-size: 0.8em` — text: "⚠️ Name erforderlich — bitte einen Namen eingeben" |
| Save button | `.btn.btn-primary` | Label: "Kreatur speichern" |
| Cancel button | `.btn` | Standard cancel |

### 6. Mobile Layout (320px)
When `[data-layout="mobile"]` is active:
- `.bestiary-layout` switches to single-column: `grid-template-columns: 1fr`.
- `.bestiary-detail` becomes full-width below the list (not a side panel).
- If a monster is selected, list collapses (display none) and detail takes full viewport width.
- A back button (`← Liste`) appears above detail panel to return to list.
- Toolbar wraps: search on first line, filters on second line (flex-wrap).
- Minimum touch target 44px enforced (inherited from existing mobile variables).

## States and Interactions

### List States

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (no selection) | Initial tab open | Full list rendered; detail panel shows empty state (dragon emoji + "Wähle ein Monster...") |
| Item hovered | Mouse over list row | `background: var(--bg-elevated)` |
| Item selected | Click list row | `border: 2px solid var(--gold); background: var(--bg-elevated)`; detail panel populates |
| Filter active | Chip toggled or dropdown changed | Chip turns gold; list filters in real-time (no debounce needed for <200 items; debounce 150ms for typed search) |
| Empty search results | No monsters match filters | Replace list with centered text "Keine Monster gefunden" |
| Favorite toggled | Click ★/☆ star | Star switches between `☆` (dim) and `★` (gold); `D.bestiaryFavorites` updated; list re-renders |

### Detail Panel States

| State | Trigger | Visual |
|-------|---------|--------|
| Empty | No monster selected | Dragon icon + "Wähle ein Monster aus der Liste" centered in panel |
| SRD monster selected | Click SRD list item | Parchment statblock; "Zur Initiative" + "Zu Encounter" buttons; NO portrait, NO edit/delete |
| Custom monster selected | Click Eigen list item | Parchment statblock; portrait (if set) in header; "Zur Initiative" + "Zu Encounter" + "Bearbeiten" + "Löschen" buttons |
| Clickable dice hover | Mouse over `+4` or `1d6+2` | Color changes to `var(--gold)`; cursor pointer; tooltip "Würfeln" |

### Editor Modal States

| State | Trigger | Visual |
|-------|---------|--------|
| Create mode | Click "+ Neue Kreatur" | Modal opens; title "Neue Kreatur"; all fields empty; no portrait shown initially |
| Edit mode | Click "Bearbeiten" on custom monster | Modal opens pre-populated with monster data; title "Kreatur bearbeiten" |
| Validation error | Save clicked without name | Red inline error "⚠️ Name erforderlich — bitte einen Namen eingeben" below name field; modal stays open |
| Save success | Valid form submitted | Modal closes; list re-renders; toast "Kreatur gespeichert" |
