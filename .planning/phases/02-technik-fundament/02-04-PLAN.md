<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backup-Datei (Dateisystem) -> App-State | Restore liest eine .json aus dem Ordner; Inhalt nicht vertrauenswuerdig |
| Dateiname -> getFileHandle | Kampagnenname fliesst in Dateinamen; Path-Traversal-Risiko |
| Dateiname -> Backup-Browser-UI | Dateinamen werden in innerHTML gerendert (XSS) |
| File System Access Permission | Ordner-Schreibzugriff; Permission-Missbrauch/Abbruch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Tampering | Path-Traversal via Kampagnenname in getFileHandle | mitigate | getBackupFilenames bereinigt auf Whitelist [a-z0-9aeoeuess-]; Unit-Test prueft kein '../' (Task 2) |
| T-02-13 | Tampering/XSS | Dateiname in Backup-Browser-Liste (persistierter XSS) | mitigate | esc() fuer alle Dateinamen/Ordnernamen in der UI (Task 3 acceptance_criteria) |
| T-02-14 | Tampering | Manipulierte Backup-Datei beim Restore | mitigate | JSON.parse in try/catch + Struktur-/Sanitize-Pruefung vor Object.assign; saveUndoState davor (Undo moeglich) |
| T-02-15 | Denial of Service | Toast-Gewitter bei wiederholten Backup-Fehlern (Spieltisch-Stoerung) | mitigate | EINE Sitzungs-Warnung (D-16 _fileBackupPausedNotified), danach stiller Pausiert-Status + Event-Log |
| T-02-16 | Tampering | Stiller Datenverlust beim Restore | mitigate | confirm-Dialog + saveUndoState VOR Anwenden (Task 3); roter Bestaetigungsdialog (UI-SPEC laut-nur-bei-Datenverlust) |
| T-02-SC | Tampering | npm/pip/cargo-Installs | accept | Keine Paket-Installs (Browser-native File System Access API). Package-Legitimacy-Gate nicht anwendbar |
</threat_model>
