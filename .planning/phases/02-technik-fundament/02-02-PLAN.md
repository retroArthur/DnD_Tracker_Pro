<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Pages (Origin) -> Browser | Ausgelieferte App + SW kommen ueber HTTPS; veralteter SW koennte dauerhaft alte App ausliefern |
| Service Worker Cache | SW kontrolliert alle Origin-Responses (Cache-Poisoning-/Stale-Klasse) |
| Update-Mechanik -> laufende Session | Erzwungenes Update koennte Session unterbrechen (D-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Denial of Service | SW liefert dauerhaft veraltete App (CACHE_VERSION nie gebumpt) | mitigate | build.py ersetzt CACHE_VERSION beim Production-Build mit versions-/timestamp-abhaengigem Wert (Task 2, Pitfall 5); Update-Hinweis D-03 |
| T-02-05 | Tampering | self.skipWaiting() im Install -> alter und neuer Code laufen gleichzeitig, Datenverlust-Risiko | mitigate | skipWaiting NUR im SKIP_WAITING-Message-Handler, nie im Install (Task 1 acceptance_criteria prueft Abwesenheit); registerServiceWorker (Plan 02-01) erzwingt keinen reload |
| T-02-06 | Spoofing/Tampering | Manifest/Icons fehlerhaft -> Install-Prompt feuert nicht oder laedt falsches Asset | mitigate | Maskable-Pruefung + DevTools-Manifest-Validierung im human-verify-Checkpoint (Task 3); echte PNGs 192+512 |
| T-02-07 | Information Disclosure | Google-Fonts-CDN-Aufruf leakt IP/Referrer (DSGVO) | mitigate | Fonts lokal gebuendelt, alle CDN-Links entfernt (Task 1 acceptance_criteria prueft Abwesenheit von fonts.googleapis.com) |
| T-02-SC | Tampering | npm/pip/cargo-Installs | accept | Keine Paket-Installs; nur offizielle GitHub-Actions (actions/deploy-pages@v4 etc., versionsgepinnt) und Python-Stdlib urllib. Package-Legitimacy-Gate nicht anwendbar |
</threat_model>
